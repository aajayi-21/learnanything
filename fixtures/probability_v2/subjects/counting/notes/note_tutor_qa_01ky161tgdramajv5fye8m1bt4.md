# Why is this true?

**Q (reader):** Why is this true?

**A:** Think of each complete outcome as a pair \((A,B)\), rather than as a timeline.

1. There are \(a\) possible choices for the \(A\)-part.
2. For each one, there are \(b\) possible choices for the \(B\)-part.
3. A tree diagram therefore has \(a\) groups of \(b\) leaves, so it has \(ab\) leaves altogether.

Nothing in that counting argument depends on which experiment happens first. The labels “A” and “B” simply identify the two parts of each combined outcome; the multiplication rule counts all compatible pairs. That is why chronological order is unnecessary.
